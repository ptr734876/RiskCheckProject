# Парсер Росреестра

Отдельный сервис: принимает координаты, открывает публичную
кадастровую карту НСПД в headless-браузере и возвращает сведения
об участке.

Вынесен в отдельный процесс, потому что Playwright поднимает
Chromium — это тяжело и медленно (секунды на запрос), и держать
это внутри основного бэкенда нельзя.

## Установка

```bash
cd frontend/src/utils/parser
python3 -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -r requirements.txt
playwright install chromium      # обязательно: скачивает браузер
```

Шаг `playwright install chromium` пропустить нельзя — без него
сервис запустится, но каждый запрос будет падать.

## Запуск

```bash
python main.py
```

Сервис поднимется на `http://127.0.0.1:8000`.
Проверка: `curl http://127.0.0.1:8000/health`

## Проверка вручную

```bash
python post.py
```

## Как связан с сайтом

Основной бэкенд (`atlas-sales-backend`) обращается сюда при поиске
адреса и подставляет полученные данные в карточку «Юридические
данные» на Шаге 1.

Связь необязательная: если парсер не запущен, сайт продолжает
работать — карта и «Факты об окружении» не зависят от него,
а в юридической карточке появится предупреждение.

Отключить обращение к парсеру: `ROSREESTR_PARSER_ENABLED=0`
в `.env` основного бэкенда.

## Ограничения

Парсер зависит от вёрстки сайта НСПД. Если там изменят селекторы,
парсер перестанет находить данные — это нормальный риск любого
парсинга. Неизвестные поля сохраняются в `extra`, чтобы при
изменении названий данные не терялись молча.

## Настройка таймаутов

Сайт НСПД тяжёлый и часто отвечает медленно. Если в логах парсера
появляется `Timeout ... exceeded` при загрузке страницы, увеличьте
таймауты через переменные окружения:

```bash
PARSER_NAV_TIMEOUT=90000 python main.py
```

| Переменная | По умолчанию | Что задаёт |
|---|---|---|
| `PARSER_NAV_TIMEOUT` | 60000 | загрузка карты НСПД, мс |
| `PARSER_UI_TIMEOUT` | 20000 | появление элементов формы, мс |
| `PARSER_RESULT_TIMEOUT` | 25000 | ожидание результатов поиска, мс |
| `PARSER_NAV_RETRIES` | 2 | попыток загрузить страницу |

## Диагностика

Парсер возвращает разные коды, по ним понятна причина:

- **404** — участок по координатам не найден. Нормальный ответ.
- **503** — сайт НСПД не отвечает или изменил вёрстку.
- **500** — прочие сбои.

Текст причины приходит в поле `detail` и показывается на сайте
в карточке «Юридические данные».

Если ошибки повторяются, первым делом откройте
https://nspd.gov.ru/map в обычном браузере. Если сайт не грузится
и там — парсер ни при чём, нужно подождать.
